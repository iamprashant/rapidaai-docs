## Authentication

You can connect to Rapida using either **Personal Tokens** (for administration APIs) or **Project API Keys** (for typical app integration).

### Administration

Personal tokens are used for APIs that require write or administrative access. These tokens are essential for managing users, roles, projects, and performing create or update operations that need authentication.

#### Implementation Example:

<CodeGroup>
  ```typescript React/Node.js

import { ConnectionConfig } from "rapida-react"; // or "rapida-nodejs"

export let connectionConfig = new ConnectionConfig();

connectionConfig.withAuth(
ConnectionConfig.WithPersonalToken({
    authorization: process.env.RAPIDA_PERSONAL_TOKEN!,
    userId: process.env.RAPIDA_USER_ID!,
    projectId: process.env.RAPIDA_PROJECT_ID!,
}));

````

```go Go
package main

import (
    "os"
    sdk "github.com/rapidaai/rapida-go"
)

func main() {
    cfg := sdk.NewConnectionConfig()
    cfg.WithAuth(sdk.WithPersonalToken(sdk.PersonalTokenOptions{
        Authorization: os.Getenv("RAPIDA_PERSONAL_TOKEN"),
        UserID:       os.Getenv("RAPIDA_USER_ID"),
        ProjectID:    os.Getenv("RAPIDA_PROJECT_ID"),
    }))
}
````

```python Python
from rapida_sdk import ConnectionConfig, WithPersonalToken
import os

connection_config = ConnectionConfig()
connection_config.with_auth(WithPersonalToken(
    authorization=os.getenv("RAPIDA_PERSONAL_TOKEN"),
    user_id=os.getenv("RAPIDA_USER_ID"),
    project_id=os.getenv("RAPIDA_PROJECT_ID"),
))
```

</CodeGroup>

### Project Credentials

Use this for typical integrations inside your apps and servers.

<CodeGroup>
  ```typescript React/Node.js
  import { ConnectionConfig } from "rapida-react"; // or "rapida-nodejs"

export let connectionConfig = new ConnectionConfig();

connectionConfig.withAuth(
    ConnectionConfig.WithSDK({
        apiKey: process.env.RAPIDA_API_KEY!,
    })
);

````

```go Go
package main

import (
    "os"
    sdk "github.com/rapidaai/rapida-go"
)

func main() {
    cfg := sdk.NewConnectionConfig()
    cfg.WithAuth(sdk.WithSDK(sdk.SDKOptions{
        ApiKey: os.Getenv("RAPIDA_API_KEY"),
    }))
}
````

```python Python
from rapida_sdk import ConnectionConfig, WithSDK
import os

connection_config = ConnectionConfig()
connection_config.with_auth(WithSDK(api_key=os.getenv("RAPIDA_API_KEY")))
```

</CodeGroup>

## Environment Variables

Create a `.env` file in your project root and add your Rapida credentials:

```bash
# For Personal Token authentication
RAPIDA_PERSONAL_TOKEN=your_personal_token_here
RAPIDA_USER_ID=your_user_id_here
RAPIDA_PROJECT_ID=your_project_id_here

# For SDK API Key authentication
RAPIDA_API_KEY=your_api_key_here
```

<Info>
  You can find your credentials in the Rapida dashboard under **Settings â†’ API
  Keys**.
</Info>

## Quick Start Example

Here's a simple example to verify your installation:

<CodeGroup>
  ```typescript Node.js
  import { ConnectionConfig } from "rapida-nodejs";

const connectionConfig = new ConnectionConfig();
connectionConfig.withAuth(
ConnectionConfig.WithSDK({
apiKey: process.env.RAPIDA_API_KEY!,
})
);

// Your Rapida integration code here
console.log("Rapida SDK initialized successfully!");

````

```go Go
package main

import (
    "fmt"
    "os"
    sdk "github.com/rapidaai/rapida-go"
)

func main() {
    cfg := sdk.NewConnectionConfig()
    cfg.WithAuth(sdk.WithSDK(sdk.SDKOptions{
        ApiKey: os.Getenv("RAPIDA_API_KEY"),
    }))

    fmt.Println("Rapida SDK initialized successfully!")
}
````

```python Python
from rapida_sdk import ConnectionConfig, WithSDK
import os

connection_config = ConnectionConfig()
connection_config.with_auth(WithSDK(api_key=os.getenv("RAPIDA_API_KEY")))

print("Rapida SDK initialized successfully!")
```

</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Error: Module not found">
    This may be due to an incorrect package name or version.

    **Solution**:
    1. Verify you're using the correct package name for your platform
    2. Check that you have the minimum required version of Node.js, Go, or Python
    3. Try reinstalling the package

  </Accordion>

  <Accordion title="Authentication errors">
    This usually indicates invalid or missing credentials.

    **Solution**:
    1. Verify your API keys are correct in your `.env` file
    2. Check that your environment variables are being loaded properly
    3. Ensure you're using the right authentication method (Personal Token vs SDK API Key)

  </Accordion>

  <Accordion title="Go module issues">
    If you encounter Go module or dependency issues.

    **Solution**:
    1. Ensure Go modules are enabled: `go mod init your-project-name`
    2. Run `go mod tidy` to clean up dependencies
    3. Check that you're using Go 1.19 or higher

  </Accordion>

  <Accordion title="Node.js version compatibility">
    If you encounter issues with Node.js compatibility.

    **Solution**:
    1. Upgrade to Node.js 16.0.0 or higher
    2. Clear your npm cache: `npm cache clean --force`
    3. Remove `node_modules` and reinstall: `rm -rf node_modules && npm install`

  </Accordion>
</AccordionGroup>
