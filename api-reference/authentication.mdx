Use `ConnectionConfig` to configure authentication and endpoints for all SDK clients.

<CodeGroup>

```ts TypeScript / Node.js
import { ConnectionConfig } from 'rapida-nodejs';

const clientCfg = new ConnectionConfig();
clientCfg.withAuth(
  ConnectionConfig.WithSDK({
    apiKey: process.env.RAPIDA_API_KEY!,
  })
);

// Optionally point to local endpoints and enable debug
clientCfg.withLocal();
// Or customize endpoints
clientCfg.withCustomEndpoint({
  assistant: 'https://assistant.api',
  web: 'https://web.api',
  endpoint: 'https://endpoint.api',
}, true);
```

```javascript React
import { ConnectionConfig } from 'rapida-react';

export const clientCfg = new ConnectionConfig()
  .withAuth(ConnectionConfig.WithSDK({ apiKey: process.env.RAPIDA_API_KEY }))
  .withLocal();
```

```go Go
package main

import (
  "os"
  sdk "github.com/rapidaai/rapida-go"
)

func main() {
  cfg := sdk.NewConnectionConfig()
  cfg.WithAuth(sdk.WithSDK(sdk.SDKOptions{ ApiKey: os.Getenv("RAPIDA_API_KEY") }))
  // cfg.WithLocal() // if supported in your SDK version
}
```

```python Python
from rapida_sdk import ConnectionConfig, WithSDK
import os

connection_config = ConnectionConfig()
connection_config.with_auth(WithSDK(api_key=os.getenv("RAPIDA_API_KEY")))
```

</CodeGroup>


####  PersonalToken
- Use for backend-only or administrative operations where a human- or service-owned token is appropriate (e.g., CI/CD, provisioning, migrations).
- Best when you need elevated privileges across projects or organization-level APIs.
- Never embed in client-side code. Store securely (vault/secret manager) and rotate regularly.

  <ResponseField name="authorization" type="string">
    Personal access token for administrative APIs.
  </ResponseField>
  <ResponseField name="userId" type="string">
    Rapida user ID performing the action.
  </ResponseField>
  <ResponseField name="projectId" type="string">
    Rapida project ID for scoping requests.
  </ResponseField>

#### WithSDK
- Use for typical application integrations (server-side web apps, backend services) using a project-scoped API key.
- Safer to use in application servers than personal tokens; permissions are limited to the project.
- Do not ship in public client apps without a proxy; prefer server-to-server.


  <ResponseField name="apiKey" type="string">
    Project API key used for typical app integrations.
  </ResponseField>
  <ResponseField name="userId" type="string">
    Optional user identifier to propagate in requests.
  </ResponseField>

#### WithWebpluginClient
- Use when embedding the Rapida web widget in a browser context.
- Keys may be short-lived or restricted for client exposure; pair with backend verification when possible.

  <ResponseField name="apiKey" type="string">
    API key for web widget usage.
  </ResponseField>
  <ResponseField name="userId" type="string">
    Optional user identifier when using the web plugin.
  </ResponseField>
